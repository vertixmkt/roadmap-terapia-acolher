Pré-requisitos
O que precisamos de você antes de começar

Para iniciar o trabalho, os itens abaixo são indispensáveis. Sem eles não é possível avançar.

Item Descrição Prioridade
Acesso ao repositório Link do GitHub/GitLab com permissão de colaborador CRÍTICO
Ambiente de produção URL + credenciais de admin do sistema atual CRÍTICO
Banco de dados Credenciais MySQL/TiDB (ou dump para ambiente de dev) CRÍTICO
ManyChat API Key + acesso à conta para mapear flows e custom fields CRÍTICO
Kiwify API Key + acesso para mapear produtos, ofertas e webhooks ALTO
Variáveis de ambiente Arquivo .env de produção ou listagem das variáveis CRÍTICO
Hospedagem Acesso ao painel onde o sistema está deployado (Manus Hosting) ALTO
Google Cloud / AI Studio Acesso para verificar integração de IA usada no matching MÉDIO
🔒 Todos os acessos serão tratados com total confidencialidade. Recomendamos criar credenciais específicas para desenvolvimento, sem comprometer as credenciais principais de produção.
Diagnóstico
Estado atual do sistema

Com base no relatório fornecido, identificamos os seguintes problemas.

Problemas Críticos
  • 742 pacientes sem terapeuta atribuído — fila parada
  • 4.798 atribuições com terapeuta inativo — dado sujo em produção
  • Links públicos de cadastro não funcionam — bloqueando entrada de novos terapeutas
Problemas Funcionais
  • Dashboard não apresenta informações úteis
  • Subscriber ID do terapeuta no ManyChat não é capturado automaticamente
  • Não existe área de acesso para o terapeuta acompanhar seu perfil
  • Validação de compra Kiwify antes de liberar cadastro não implementada
Código Legado (será removido)
  • Integração Z-API (abandonada, derrubava chips)
  • Telas: Monitoramento Fila, Análise Mensalidades
  • Telas: Métricas Terapeuta, Comparação Envios
  • Telas: Auditoria, Feedback, Debug Logs, Histórico Z-API
Execução
Roadmap em 6 Fases

Cada fase tem objetivo claro, lista de entregas e um entregável concreto ao final.

0
Onboarding e Setup
Semana 1
Diagnóstico
Entender o código de verdade e configurar ambiente de desenvolvimento local.
  • Receber e validar todos os acessos listados
  • Mapear estrutura real do repositório
  • Subir ambiente local de desenvolvimento
  • Revisar banco de dados — tabelas, índices, dados reais
  • Mapear todos os flows do ManyChat e custom fields em uso
  • Mapear webhooks ativos da Kiwify
  • Auditoria de telas ativas x telas mortas
  • Documentar algoritmo de matching atual (código real, não relatório)
📄 Entregável: Relatório de diagnóstico técnico com prioridades validadas e aprovadas pelo cliente
1
Limpeza e Estabilização
Semana 2
Limpeza
Sistema mais leve, sem código morto, sem bugs críticos em produção.
  • Remover telas e código morto (Z-API e telas não utilizadas)
  • Remover routers e procedures de Z-API do backend
  • Investigar causa raiz dos 742 pacientes sem terapeuta
  • Corrigir lógica de matching para evitar fila parada
  • Tratar as 4.798 atribuições com terapeuta inativo (reprocessar ou arquivar — definir com cliente)
  • Criar script de auditoria automática para detectar casos futuros
✅ Entregável: Sistema sem código morto, banco limpo e deploy em produção com rollback documentado
2
Correções Funcionais
Semanas 3 e 4
Funcionalidades
Tudo que existe no sistema funciona de verdade.
  • Dashboard redesenhado com métricas úteis e ações rápidas
  • Links públicos de cadastro de paciente e terapeuta funcionando
  • Validação de compra Kiwify antes de liberar cadastro de terapeuta
  • Captura automática do Subscriber ID do terapeuta via ManyChat
  • Interface para cadastrar produtos Kiwify com quantidade de leads
  • Saldos de leads atualizando automaticamente via webhook
🚀 Entregável: Dashboard funcional, links operando, saldos automáticos, cadastro validado
3
Portal do Terapeuta
Semana 5
Nova Funcionalidade
O terapeuta tem autonomia para gerenciar seus próprios dados sem precisar do admin.
  • Login via link único enviado pelo ManyChat (token temporário)
  • Tela "Meu Perfil" — editar dados, turnos, abordagem, público atendido
  • Tela "Minhas Atribuições" — lista de pacientes recebidos e histórico
  • Tela "Meu Saldo" — histórico de compras, saldo atual, link para comprar mais
  • Restrição total — terapeuta só acessa seus próprios dados
👤 Entregável: Portal do terapeuta funcional, seguro e com autenticação separada da área admin
4
Algoritmo de Matching
Semana 6
Inteligência
Matching mais preciso, com controle fácil pelo admin — sem precisar de desenvolvedor para ajustar.
  • Auditoria completa do algoritmo real (código-fonte, não documentação)
  • Correção dos casos onde pacientes ficam presos sem atribuição
  • Painel de controle — ajustar pesos dos critérios (gênero, turno, abordagem, IA)
  • Log visual de cada decisão do algoritmo — por que X foi atribuído a Y
  • Matching manual com seleção de terapeuta pelo admin
🧠 Entregável: Algoritmo revisado, zero pacientes presos, admin controla critérios sem precisar de dev
5
Polimento Final
Semana 7
Entrega
Sistema pronto para operar de forma autônoma, sem suporte técnico constante.
  • Revisão de UX das telas de Pacientes, Terapeutas e Atribuições
  • Exportação de atribuições para Excel com filtros por período e status
  • Logs ManyChat com re-envio fácil em caso de erro
  • Testes de carga com 10k+ pacientes e múltiplos usuários simultâneos
  • Documentação de operação para o time administrativo
🎯 Entregável: Sistema estável, documentado e time treinado para operar sem suporte técnico
