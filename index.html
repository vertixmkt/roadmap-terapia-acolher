01 Pré-requisitos

O que precisamos
antes de começar

Para iniciar o trabalho, os itens abaixo são indispensáveis. Sem eles não é possível avançar.

Item Descrição Prioridade
Repositório Link do GitHub/GitLab com permissão de colaborador CRÍTICO
Ambiente de produção URL + credenciais de admin do sistema atual CRÍTICO
Banco de dados Credenciais MySQL/TiDB ou dump para ambiente de dev CRÍTICO
ManyChat API Key + acesso à conta para mapear flows e custom fields CRÍTICO
Kiwify API Key + acesso para mapear produtos, ofertas e webhooks ALTO
Variáveis de ambiente Arquivo .env de produção ou listagem completa CRÍTICO
Hospedagem Acesso ao painel onde o sistema está deployado (Manus Hosting) ALTO
Google Cloud / AI Studio Acesso para verificar integração de IA usada no matching MÉDIO
🔒

Todos os acessos serão tratados com total confidencialidade. Recomendamos criar credenciais específicas para o desenvolvimento, sem comprometer as credenciais principais de produção.

02 Diagnóstico

Estado atual
do sistema

Com base no relatório fornecido, identificamos os seguintes problemas ativos.

Problemas Críticos
  • 742 pacientes sem terapeuta atribuído — fila parada
  • 4.798 atribuições com terapeuta inativo — dado sujo em produção
  • Links públicos de cadastro não funcionam — bloqueando entrada de novos terapeutas
Problemas Funcionais
  • Dashboard não apresenta informações úteis
  • Subscriber ID do terapeuta não é capturado automaticamente
  • Não existe área de acesso para o terapeuta acompanhar perfil e atribuições
  • Validação de compra Kiwify antes de liberar cadastro não implementada
Legado (será removido)
  • Integração Z-API (abandonada, derrubava chips)
  • Monitoramento Fila, Análise Mensalidades
  • Métricas Terapeuta, Comparação Envios
  • Auditoria, Feedback, Debug Logs, Histórico Z-API
03 Roadmap de Execução

6 Fases,
2 semanas

Cada fase tem objetivo claro, lista de entregas e um entregável concreto ao final.

0

Onboarding e Setup

Semana 1 · Dias 1–2
Diagnóstico

Entender o código de verdade e configurar ambiente de desenvolvimento local.

  • Receber e validar todos os acessos listados
  • Mapear estrutura real do repositório e subir ambiente local
  • Revisar banco de dados — tabelas, índices, dados reais
  • Mapear flows do ManyChat, custom fields e webhooks da Kiwify
  • Auditoria de telas ativas x telas mortas e algoritmo de matching
📄Entregável: Relatório de diagnóstico técnico com prioridades validadas
1

Limpeza e Estabilização

Semana 1 · Dias 3–4
Limpeza

Sistema mais leve, sem código morto, sem bugs críticos em produção.

  • Remoção de telas e código morto (Z-API e telas não utilizadas)
  • Investigar causa raiz dos 742 pacientes sem terapeuta e corrigir
  • Tratar as 4.798 atribuições com terapeuta inativo
  • Script de auditoria automática para detectar casos futuros
✅Entregável: Sistema sem código morto, banco limpo, deploy com rollback documentado
2

Correções Funcionais

Semana 1 · Dias 5–7
Funcionalidades

Tudo que existe no sistema funciona de verdade.

  • Dashboard redesenhado com métricas úteis e ações rápidas
  • Links públicos de cadastro de paciente e terapeuta funcionando
  • Validação de compra Kiwify antes de liberar cadastro de terapeuta
  • Captura automática do Subscriber ID do terapeuta via ManyChat
  • Interface para produtos Kiwify + saldos atualizando via webhook
  • Migração e otimização do banco de dados
  • Integração com novos canais além de ManyChat e Kiwify
🚀Entregável: Dashboard funcional, links operando, banco migrado, novas integrações ativas
3

Portal do Terapeuta

Semana 2 · Dias 8–9
Nova Feature

O terapeuta tem autonomia para gerenciar seus próprios dados sem precisar do admin.

  • Login via link único enviado pelo ManyChat (token temporário)
  • Tela "Meu Perfil" — editar dados, turnos, abordagem, público atendido
  • Tela "Minhas Atribuições" — lista de pacientes e histórico
  • Tela "Meu Saldo" — histórico de compras, saldo atual, link para mais
  • Aplicativo mobile (iOS e Android) para acesso do terapeuta
  • Videochamada integrada entre terapeuta e paciente
  • Chat interno entre terapeuta e paciente
👤Entregável: Portal web + app mobile com videochamada e chat integrados
4

Algoritmo de Matching

Semana 2 · Dia 10
Inteligência

Matching mais preciso com controle fácil pelo admin — sem precisar de desenvolvedor.

  • Auditoria completa do algoritmo real (código-fonte)
  • Correção dos casos onde pacientes ficam presos sem atribuição
  • Painel de controle — ajustar pesos dos critérios (gênero, turno, IA)
  • Log visual de cada decisão do algoritmo
  • Matching manual com seleção de terapeuta pelo admin
🧠Entregável: Algoritmo revisado, zero pacientes presos, admin controla sem dev
5

Polimento Final

Semana 2 · Dias 11–14
Entrega

Sistema pronto para operar de forma autônoma, sem suporte técnico constante.

  • Redesign visual completo do sistema
  • Revisão de UX das telas de Pacientes, Terapeutas e Atribuições
  • Exportação de atribuições para Excel com filtros
  • Logs ManyChat com re-envio fácil em caso de erro
  • Testes de carga com 10k+ pacientes e múltiplos usuários simultâneos
  • Documentação de operação para o time administrativo
🎯Entregável: Sistema estável, novo visual, documentado e time treinado
